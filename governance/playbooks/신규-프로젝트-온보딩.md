# 플레이북: 신규 자동화 프로젝트 합류

새 자동화 프로젝트를 거버넌스 시스템에 등록하는 표준 절차.

## 트리거

- 새 자동화 프로젝트 코드를 작성하기 시작할 때
- 기존 프로젝트를 거버넌스에 합류시킬 때

## 사전 결정 사항

| 항목 | 결정 |
|---|---|
| 프로젝트 이름 | 한글 OK, 영문 권장 (인코딩 안전) |
| 위치 | `D:\<이름>` (D 루트 직접) |
| 타입 | governance / tool / automation / docs / docs-output / media / query |
| 백업 방식 | zip / mirror / skip |
| 자체 스케줄러 보유? | true → `self_scheduled: true` 표시 |
| NotebookLM 필수? | true / false |
| `/review` 포함? | true / false |

## 7단계 절차

### 1. 프로젝트 폴더 생성

```powershell
mkdir D:\<이름>
cd D:\<이름>
```

스타터 템플릿 사용 시 (Phase 6):
```powershell
Copy-Item D:\메인작업\templates\automation-skeleton\* D:\<이름>\ -Recurse
```

### 2. 프로젝트 자체 CLAUDE.md 작성

- 150줄 이내 (`policies/문서화.md` 규칙)
- 첫 줄: 프로젝트 한 줄 소개
- 섹션: 개요·entry·사용법·주의사항

### 3. `manifest.yaml` 등록

`D:\메인작업\manifest.yaml` `projects:` 섹션에 항목 추가:
```yaml
- name: <이름>
  path: D:\<이름>
  status: active
  purpose: ...
  type: ...
  entry: ...
  schedule: { type: ..., trigger: ... }
  self_scheduled: false
  review_enabled: ...
  notebooklm_required: ...
  backup_method: zip
  tags: [...]
```

### 4. 백업 대상이라면 backup_projects.ps1 동기화

`D:\백업도구\backup_projects.ps1` 의 `$zipFolders` (또는 `$mirrorFolders`)에 경로 추가.
`policies/백업.md` 본문도 갱신 (대상 목록 표).

### 5. /review 대상이라면 review_targets.json 동기화

```powershell
python D:\메인작업\tools\sync_review_targets.py --dry-run
python D:\메인작업\tools\sync_review_targets.py
```

또는 수동: `~/.claude/review_targets.json` 의 `projects` 에 항목 추가.

### 6. 슬래시커맨드 필요 시 등록

`~/.claude/commands/<이름>.md` 작성 + `policies/슬래시-커맨드.md` 표 갱신 + `manifest.yaml` `slash_commands` 섹션 추가.

### 7. 거버넌스 git commit + push

```powershell
cd D:\메인작업
git add manifest.yaml policies/백업.md policies/슬래시-커맨드.md
git commit -m "feat(<이름>): 신규 프로젝트 합류"
git push
```

## 검증

- [ ] D:\<이름>\CLAUDE.md 존재
- [ ] manifest.yaml 등록 완료
- [ ] (백업 대상이면) 다음 매월 1일 백업 시 ZIP 생성됨
- [ ] (review 대상이면) `/review` 또는 `/full-review`에서 인식

## 관련 자료

- 정책: `policies/문서화.md`, `policies/백업.md`, `policies/슬래시-커맨드.md`
- 템플릿: `templates/automation-skeleton/`
- 도구: `tools/sync_review_targets.py`
