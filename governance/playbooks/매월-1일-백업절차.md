# 플레이북: 매월 1일 백업 절차

매월 1일 09:00에 자동 실행되는 백업의 전체 흐름·검증·복구 절차.

## 트리거

자동: Windows Task Scheduler (`프로젝트백업_매월1일`) 매월 1일 09:00.
수동: 필요 시 PowerShell에서 직접 실행.

## 사전 조건

- D 드라이브 여유 공간 ≥ 15GB
- C 드라이브 여유 공간 ≥ 10GB (Claude Code 기본 동작 보장)
- `D:\백업도구\backup_projects.ps1` 존재 + UTF-8 BOM 저장 상태
- 백업 대상 폴더 9개 + 영상편집 1개 모두 D:\ 루트 존재 (`manifest.yaml` 정본 기준)

## Task Scheduler 등록 (1회)

```powershell
schtasks /Create /XML "D:\백업도구\backup_task.xml" /TN "프로젝트백업_매월1일"
```

검증:
```powershell
schtasks /Query /TN "프로젝트백업_매월1일" /V /FO LIST | Select-Object -First 20
```

## 수동 실행

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File "D:\백업도구\backup_projects.ps1"
```

## 실행 후 검증 (백업-무결성-검사 에이전트 위임)

`agents/백업-무결성-검사.md` 호출. 5단계 검사:
1. ZIP 9개 존재
2. 크기 sanity check (이전 월 ±50%)
3. ZIP 무결성 (`tar -tf`)
4. backup_log.txt ERROR/FAIL 카운트
5. 영상편집 미러 비교

## 실패 시 복구

| 증상 | 원인 | 조치 |
|---|---|---|
| ZIP 9개 미생성 | C/D 디스크 풀 | `skills/풀-디스크-사고-회피.md` 절차 |
| 한글 파일명 깨짐 (robocopy) | CP949↔UTF-16 변환 | `skills/한글-파일명-안전처리.md` 사례 2 — Copy-Item 또는 7z 검토 |
| ps1 파서 에러 (`Unexpected token`) | UTF-8 BOM 누락 | `skills/한글-파일명-안전처리.md` 사례 1 — BOM 재저장 |
| Task Scheduler 09:00 미실행 | PC 꺼짐 | `<StartWhenAvailable>true</StartWhenAvailable>`로 깨어났을 때 보충 (이미 적용) |
| schtasks "Access is denied" | 권한 부족 | 관리자 권한 PowerShell에서 재등록 |
