# 플레이북: 프로젝트별 알림 통합

자동화 프로젝트가 산출물(보고서·엑셀·영상·문서)을 만들었을 때 박광우님 화면 우하단에 **Windows 토스트 알림**을 띄우게 만드는 절차.

## 트리거

신규 프로젝트가 거버넌스에 합류한 직후, 또는 기존 프로젝트의 산출물이 "조용히 생성"되어 박광우님이 인지 못 하는 경우 발견 시.

## 사전 조건

- 공통 도구 `D:\메인작업\tools\notify.py` 존재 (이미 작성 완료)
- 대상 프로젝트 폴더에 진입한 별도 Claude Code 세션
  - 메인작업 세션은 거버넌스 차단 훅 때문에 다른 프로젝트 폴더 수정 불가
  - 따라서 이 플레이북은 각 프로젝트 세션에서 실행
- 박광우님 PC에 Python + pip 사용 가능 (모든 프로젝트 공통 전제)
- `winotify` 패키지: 첫 통합 프로젝트에서 1회 설치 (이후 모든 프로젝트가 공유)

## Step 1 — winotify 설치 (1회만)

처음 통합하는 프로젝트의 가상환경(또는 시스템 Python)에서:

```powershell
pip install winotify
```

이후 다른 프로젝트는 자기 가상환경에서 동일하게 `pip install winotify`. `requirements.txt`가 있는 프로젝트는 `winotify>=1.1,<2.0` 한 줄 추가 권장.

## Step 2 — Python 프로젝트에서 호출 (2가지 방법)

### 방법 A: 한 줄 import (권장)

산출물 생성 함수 마지막에 추가:

```python
import sys
sys.path.insert(0, r"D:\메인작업\tools")
from notify import send_toast

send_toast(
    app_id="🏥 케어포 점검",
    title="✅ 오늘 일일 점검 완료",
    message=f"{datetime.now():%H:%M} 생성 — 클릭하면 엑셀이 열립니다.",
    file_path=str(report_path),                       # 알림 클릭 시 열림
    folder_path=str(report_path.parent),              # "📂 폴더 열기" 보조 액션
    alert_level="info",                               # "info" | "warning" | "critical"
)
```

위험 발견 시 등급 격상:

```python
risk_count = count_critical_risks(check_result)
send_toast(
    app_id="🏥 케어포 점검",
    title=f"위험 {risk_count}건 발견" if risk_count else "오늘 점검 완료",
    message="즉시 확인하세요" if risk_count else "특이사항 없음",
    file_path=str(report_path),
    folder_path=str(report_path.parent),
    alert_level="warning" if risk_count > 0 else "info",
)
```

### 방법 B: CLI 호출 (PowerShell / 배치 / 빠른 테스트)

```powershell
python "D:\메인작업\tools\notify.py" `
    --app-id "💾 백업도구" `
    --title "이번 달 백업 완료" `
    --message "ZIP 9개 + 미러 갱신" `
    --file "D:\백업도구\결과\2026-06-01\" `
    --folder "D:\백업도구\결과\" `
    --level info
```

배치 파일에서:
```bat
python "D:\메인작업\tools\notify.py" --app-id "📝 블로그" --title "오늘 게시 완료" --file "%REPORT_PATH%"
```

## Step 3 — 검증

테스트 호출:
```powershell
python "D:\메인작업\tools\notify.py" --app-id "테스트" --title "알림 작동 확인" --message "이 알림이 보이면 성공"
```

성공 신호:
- 화면 우하단에 알림 카드 표시
- 알림 센터(Windows + N)에 누적됨
- 클릭 시 (file 지정한 경우) 파일 자동 열림

실패 시:
- 아무것도 안 보임 → `pip install winotify` 다시 확인 (해당 프로젝트의 venv 활성화 상태에서)
- 에러 메시지 출력 → `python -c "from winotify import Notification; print('ok')"` 로 패키지 로드 검증

## Step 4 — 해당 프로젝트 CLAUDE.md에 한 줄 기록

해당 프로젝트의 CLAUDE.md 변경 이력에 추가:

```
- YYYY-MM-DD: 산출물 생성 직후 Windows 토스트 알림 추가 — `D:\메인작업\tools\notify.py` send_toast() 호출. 알림 클릭 시 산출물 즉시 열림. winotify 미설치 시 조용히 스킵.
```

## 트러블슈팅

| 증상 | 원인 | 해결 |
|---|---|---|
| 알림 안 보임 | winotify 미설치 | 해당 프로젝트 venv 활성화 후 `pip install winotify` |
| 알림 보이지만 클릭해도 안 열림 | file_path 경로 오류 | `Path.exists()` 로 사전 확인 후 호출 |
| 알림이 너무 자주 뜸 | 호출 빈도 높음 | 조건부 호출(에러/완료만) 또는 한 번에 묶어서 호출 |
| 자동화 자체가 멈춤 | send_toast 예외가 메인 흐름에 전파 | 도구 자체가 예외 차단함 — 호출 측 코드의 다른 부분 확인 |
| 어떤 프로젝트 알림인지 헷갈림 | app_id 일관성 부족 | 프로젝트별 고유 이모지+이름 (예: "🏥 케어포 점검") 표준화 |

## 알림 등급 가이드

| 등급 | 사용 시점 | 표시 |
|---|---|---|
| `info` | 정상 완료 | 아무 prefix 없음 |
| `warning` | 주의 필요 (위험 발견, 일부 실패) | ⚠️ |
| `critical` | 긴급 (자동화 실패, 위험 다수) | 🚨 |
