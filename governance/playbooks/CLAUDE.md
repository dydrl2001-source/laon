# playbooks/ — 표준 운영 절차

반복적으로 발생하는 작업의 단계별 지침. 사람이 따라하는 형태(Markdown)로 작성. 다른 사람이 인수인계받아도 그대로 실행 가능해야 함.

## 플레이북 분류

| 카테고리 | 예시 | 빈도 |
|---|---|---|
| 정기 운영 | 매월 1일 백업 절차 | 매월 |
| 시즌 운영 | 분기별 운영위원회 회의록 작성 | 분기 |
| 신규 합류 | 신규 프로젝트 온보딩 | ad-hoc |
| 사고 대응 | C 디스크 압박 해소 절차 | ad-hoc (학습) |
| 거버넌스 유지 | manifest → review_targets 동기화 | manifest 변경 시 |
| 인프라 운영 | GitHub 거버넌스 동기화 (commit·push·새PC clone) | 주기 또는 ad-hoc |

## 작성 원칙

1. **목적·트리거 명시** (언제 실행하는가)
2. **사전 조건** (필요한 권한, 도구, 환경)
3. **단계별 명령** (복사-붙여넣기 가능한 수준)
4. **검증 방법** (성공 신호)
5. **트러블슈팅** (자주 발생하는 에러 + 해결)
