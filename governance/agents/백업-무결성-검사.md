## 검사 대상
- ZIP: D:\백업도구\결과\<YYYY-MM-DD>\<프로젝트명>.zip × 9
- 미러: D:\백업도구\결과\_mirror\영상편집\
- 로그: D:\백업도구\결과\backup_log.txt

## 검사 항목 (5단계)

### 1. 존재 검증
- ZIP 9개 파일 존재 (운영위원회 작성·카톡보고 피드백·희망이음반자동·블로그자동화·carefor_checker_saas·메인작업·법령스마트검색·문서교정·케어포작성)
- 각 ZIP 0바이트 아님

### 2. 크기 sanity check
- 이전 월 대비 ±50% 이내 (블로그자동화·문서교정 같은 큰 폴더)
- 이상 변동 시 ⚠️

### 3. ZIP 무결성
```
tar -tf <zip> > /dev/null && echo OK || echo CORRUPT
```
- 파일명 인코딩 깨짐 검출 (한글 파일이 ?로 표시되는지)

### 4. 로그 검사
- backup_log.txt 의 ERROR / FAIL 라인 카운트
- robocopy "FAILED" 카운트 — 이전 월 대비 증가 여부
- 시작·종료 시각 기록 정상 여부

### 5. 미러 비교
- D:\영상편집 vs D:\백업도구\결과\_mirror\영상편집
- 파일 수 / 총 크기 비교 (영상파일 제외분 일치해야)

## 출력 형식
1. **종합 판정**: OK / WARNING / FAIL
2. **항목별 결과** (위 5단계 표)
3. **이상 항목 상세** (있을 때만)
4. **권고 조치** (재실행 / 무시 가능 / 즉시 수정)
