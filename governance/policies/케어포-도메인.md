## NotebookLM MCP 불통 시 Fallback

1. MCP 호출이 브라우저 오류·인증만료·timeout으로 실패하면 **즉시 자체 추측 답변 금지**
2. 로컬 평가매뉴얼 원본 직독 — 경로:
   - `C:\Users\user\Desktop\공문 및 평가와 사업\25년 평가\2025년도_시설급여_장기요양기관_평가_매뉴얼(노인요양시설).pdf`
   - `C:\Users\user\Desktop\공문 및 평가와 사업\25년 평가\2025년_시설급여_평가매뉴얼_다빈도_Q&A_사례집(2차).pdf`
3. PDF 직독 도구 우선순위:
   - 1순위: `mcp__pdf-toolkit__*` (Claude Desktop 임포트)
   - 2순위: `mcp__pdf-viewer__*` (Anthropic 공식)
   - 3순위: Read 툴 `pages: "N-M"` (페이지 범위 지정 필수, 큰 PDF는 분할)
   - **`pdftoppm` CLI 시도 금지** — 미설치 + 위 MCP/Read로 직독 가능
4. 매뉴얼 위치 모르면 사용자에게 경로 확인 — 자의적 위치 가정 금지
5. 자체 지식 단독 결론 금지 — fallback도 실패하면 "매뉴얼 미확인" 명시

## 점검 시 5가지 동시 검증

1. 규정 준수
2. 체크 ↔ 서술 일치
3. 갱신 규칙 (주간/월간/분기/반기/연간)
4. 오타
5. CIST 공란

## 문제 피드백 형식

문제 보고 시 반드시 **어르신 이름 + 구체적 문제** 명시 (일반적 표현 금지).
- ❌ "일부 어르신 체중 데이터 누락"
- ✅ "김XX 어르신 2026-04 월간 체중 측정 누락"

## 급여제공결과평가 변화 감지

- 50%+ 변화 필수 확인
- 인지기능평가 반응기록 30%+ 변화 필수 확인
- `_check_result_minimal_change()`로 0.50~0.60 구간 warn 처리

## 야간점검

- 기준: **3회 이상** (5회 아님)
- 특이사항 공란 없으면 OK
