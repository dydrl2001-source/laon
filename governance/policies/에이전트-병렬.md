# 에이전트 병렬 투입 정책

## 핵심 원칙

**점검·분석·리뷰 시 항상 전문가 에이전트 13명 동시 투입**.
근거: 정확도 + 효율 동시 향상 (사용자 명시 선호, feedback_always_use_10_agents.md).

## 적용 영역

- `/점검`: 평가지표별 도메인 전문가 + 데이터무결성 + 회귀 등
- `/review`: 5에이전트 (코드품질·보안·에러핸들링·테스트·문서화)
- `/full-review`: 13에이전트 (도메인별 + 통합 회귀)
- 분석성 사용자 질문: 다관점 에이전트 동시 투입 후 종합

## 에이전트 풀 (도메인 3 + 엔지니어급 5 = 8종)

거버넌스 repo `agents/`에 정의된 전문가 카드. 작업 성격에 따라 조합 선택.

### 도메인 (3)
- `케어포-도메인-전문가` — 평가지표·점검·운영
- `평가매뉴얼-자문` — 매뉴얼 조항 인용·해석
- `백업-무결성-검사` — 백업 결과물 검증

### 엔지니어급 메타 (5) — Anthropic 엔지니어 수준 판단
- `엔지니어-스태프-SWE` — 코드 품질·아키텍처·리팩토링 (14개 프로젝트 부채 식별)
- `엔지니어-SRE` — 자동화 신뢰성·관측성·장애 분석 (Task Scheduler·백업·정기 보고)
- `엔지니어-보안` — `.env`·토큰·.gitignore·권한 감사
- `엔지니어-AI프롬프트` — 슬래시커맨드·에이전트 정의·CLAUDE.md 메타 최적화
- `엔지니어-성능비용` — 토큰·실행시간·MCP 호출 효율

### 작업별 권장 조합
- **신규 자동화 설계**: Staff SWE + SRE + 보안 + 성능비용 (4)
- **기존 코드 리팩토링**: Staff SWE + 보안 + AI프롬프트 (3)
- **자동화 장애 대응**: SRE + Staff SWE (2)
- **거버넌스 자료(정책·에이전트·커맨드) 개선**: AI프롬프트 + 성능비용 (2)
- **분기별 `/full-review`**: 도메인 3 + 엔지니어급 5 전부 (8) + 통합 회귀

## Opus 4.7 활용 7원칙

1. **에이전트 병렬 투입** — 점검·분석·리뷰 시 전문가 13명 동시 투입
2. **단순 질문은 즉시 답변** — Adaptive Thinking 자동 판단, 불필요한 과잉분석 금지
3. **복잡한 작업은 Plan 먼저** — 다단계 구현 전 반드시 플랜 수립 후 실행
4. **컨텍스트 효율 관리** — 불필요한 파일 재읽기 금지, 한 번 읽은 정보는 즉시 활용
5. **플러그인 적극 활용** — insane-search, deep-research, 공식 스킬 우선 사용
6. **NotebookLM 우선 참조** — 평가 기준·법령은 자체 지식 대신 업로드 문서 기반
7. **진행 확인 질문 금지** — 다음 단계 바로 실행, 사용자 시간 낭비 방지
