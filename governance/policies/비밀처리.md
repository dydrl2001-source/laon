# 비밀 처리 정책

## 핵심 원칙

**모든 비밀번호·OAuth 토큰·API 키·인증서는 별도 `.env` 파일로 관리**.
- 절대 코드에 하드코딩 금지
- 절대 git 커밋 금지 (.gitignore에 명시)

## 케어포 로그인

- 케어포 아이디: `박광우`
- 비밀번호: 각 프로젝트의 `.env` 참조 (`facility "1"` = 1관 / `facility "2"` = 2관)
- 케어포 주소: https://www.carefor.co.kr

## .env 통합 금지 (영구 정책)

각 자동화 프로젝트는 자체 `.env`를 보유. 매니저가 중앙으로 통합 시도 금지.
이유: 기존 코드의 `.env` 로딩 경로 변경 시 코드 수정 불가피 → "코드 비침습" 원칙 위반.

확인된 분산 .env 위치:
- `D:\carefor_checker_saas\.env`
- `D:\블로그자동화\.env`
- `D:\희망이음반자동\.env`
- (그 외 프로젝트별)

## 인증 캐시

각 프로젝트의 `auth.json`, `chrome_profile/`, NPKI 등은 해당 프로젝트 폴더 내 유지.
매니저는 위치만 인지하고 직접 접근하지 않음.

## 커밋 안전

`.gitignore`에 `.env`, `*.key`, `credentials.json`, `token.json`, `*.pem`, `auth.json` 등록.
신규 프로젝트 합류 시 .gitignore 패턴 검토 필수.
