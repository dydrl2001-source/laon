# 문서화 정책

## CLAUDE.md 하위폴더 분리

- 프로젝트 지식은 **해당 폴더의 CLAUDE.md에 직접 기록** (별도 memory/ 파일 생성 금지)
- 루트 CLAUDE.md: 프로젝트 개요, 원칙, 현황
- 하위폴더 CLAUDE.md: 해당 모듈 전용 규칙, 버그 이력, 기술 메모
- 각 CLAUDE.md는 **150줄 이내** 유지

## 150줄 초과 시 분할 우선

오래된 세부사항 압축보다 `CLAUDE_v2.md`, `CLAUDE_legacy.md` 등 **2번째 파일로 이관**.
루트 CLAUDE.md 상단에 `@CLAUDE_v2.md` 형태로 이어읽기 링크 유지 (Claude Code가 해당 파일도 자동 로드).

분할 기준: 최근 30일 내 변경사항은 메인에, 그 이전 이력·버그 히스토리는 v2/legacy로.

## 즉시 갱신 원칙

**코드 수정 완료 직후 해당 CLAUDE.md 즉시 업데이트** — 나중에 몰아서 하지 않는다.

## 자동 상속

Claude Code가 작업 폴더 기준 상위→하위 자동 로드하므로 별도 설정 불필요.
글로벌 `~/.claude/CLAUDE.md` → 매니저 `D:\메인작업\CLAUDE.md` (import) → 프로젝트별 `.claude/CLAUDE.md`.

## 버그 수정 메모리화

코드 수정 시 반드시 원인·해결 메모리 기록 (사용자 명시 요청, 기본값).
- 위치: `~/.claude/projects/<project-id>/memory/feedback_*.md`
- 형식: 현상 → 원인 → 해결 → 검증

## 보고 전 검증

파일 생성 완료 보고 전 실제 파일 검증 필수 (복사 ≠ 생성, 과장 금지).

## 터미널 역할 분리

명시적 지시 없이 코드 작성 금지. 전략 터미널에서는 분석/정리만.
